I use Claude to help me write git commit messages all the time. I give it my diff and it gives me a message. But the quality is inconsistent — sometimes it's too vague ("fix bug"), sometimes it's too detailed. I want to turn this into a proper skill for the registry. The format I want: Conventional Commits (feat/fix/chore/etc.), one-line subject under 72 chars, optional body for non-trivial changes.
